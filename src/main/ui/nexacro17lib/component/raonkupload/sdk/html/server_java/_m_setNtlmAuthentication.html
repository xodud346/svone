K Upload :: setNtlmAuthentication

void setNtlmAuthentication(String pDomain, String pUsername, String pPassword)

파일을 업로드 하려는 위치가 계정이 있는 네트워크 드라이브이거나,
다운로드 하려는 파일의 위치가 계정이 있는 네트워크 드라이브일 경우
네트워크 드라이브의 인증 정보를 설정해야 합니다.
JAVA에서는 SMB 프로토콜을 사용하며, 역슬래시(\)가아닌 슬래시(/)로 경로설정이 되어야합니다.

return value

없음.

parameters

pDomain  Network Drive 접속 Domain을 의미합니다.
pUsername  Network Drive 인증 User Name를 의미합니다.
pPassword  Network Drive 인증 Password를 의미합니다.

remarks

1. SMB Protocol 연결 설정
SMB Protocol 사용을 위한 연결 설정을 해야 합니다.
ex) upload.settingVo.setNtlmAuthentication("smb://SMB Domain 또는 IP", "ID", "Password");
2. 파일 업로드시 사용되는 Temp Path 설정
파일 업로드시 사용되는 Temp Path를 SMB Protocol의 경로로 설정해야 합니다.
ex) upload.settingVo.setTempPath("smb://SMB Domain 또는 IP/temp ");
3. 파일이 업로드 될 최종 Path 설정
파일이 업로드 될 최종 Path를 SMB Protocol의 경로로 설정해야 합니다.
ex) upload.settingVo.setPhysicalPath("smb://SMB Domain 또는 IP/savePath");

sample code

RAONKHandler upload = new RAONKHandler();

upload.settingVo.setNtlmAuthentication("smb://SMB Domain 또는 IP", "ID", "Password");
upload.settingVo.setTempPath("smb://SMB Domain 또는 IP/temp ");
upload.settingVo.setPhysicalPath("smb://SMB Domain 또는 IP/savePath");

String result = upload.Process(request, response, application, event);
